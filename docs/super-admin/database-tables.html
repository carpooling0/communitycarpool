🗄️ Database

Tables reference

Complete reference for all database tables. RLS is enabled on all tables with no policies defined — this means the anon role has zero access, and all data operations go through edge functions using the service role key.

ℹ️
Tables marked Planned do not yet exist in the database. They are documented here for future implementation.
users Active
Stores one record per registered user. A user is created (or updated) when they first submit a journey via submit-journey.
ColumnTypeDescription
user_idSERIAL PKAuto-incrementing primary key
emailTEXT UNIQUE NOT NULLUser's email address — used as the unique identifier
first_nameTEXTFirst name as entered on the form
last_nameTEXTLast name as entered on the form
org_codeTEXTOptional organisation code — links user to an org in the organisations table
journey_tokenTEXTSecure token embedded in the matches page URL (e.g. /matches.html?token=...)
journey_token_expires_atTIMESTAMPTZToken expiry — set to created_at + match_token_expiry_days config value
last_seen_atTIMESTAMPTZLast time the user accessed their matches page
deletion_requested_atTIMESTAMPTZWhen the user initiated a deletion request — null if no pending deletion
deletion_tokenTEXTToken sent in the deletion confirmation email
deletion_token_expires_atTIMESTAMPTZDeletion token expiry (typically 24 hours)
terms_accepted_versionTEXTVersion of T&C the user last accepted (e.g. "1.1")
terms_accepted_atTIMESTAMPTZWhen the user last accepted the T&C
email_whitelistBOOLEAN DEFAULT falseWhen true, user receives emails even when testing_mode = true
created_atTIMESTAMPTZ DEFAULT now()Account creation timestamp
submissions Active
Each row is a journey submission. One user can have multiple submissions (e.g. different routes or schedules). Submissions expire after 90 days.
ColumnTypeDescription
sub_idSERIAL PKAuto-incrementing primary key
user_idINT FK → usersForeign key to the submitting user
from_locationTEXTHuman-readable home/start address (as returned by Google Maps autocomplete)
to_locationTEXTHuman-readable work/destination address
from_pointGEOGRAPHY(Point)PostGIS geography point for the start location. Stored as WKB internally — use get_submission_coords RPC to retrieve as float lat/lng.
to_pointGEOGRAPHY(Point)PostGIS geography point for the destination
distance_prefTEXTHow far the user is willing to travel to meet a carpool partner (e.g. "500m", "1km", "2km")
work_scheduleTEXTDays/times the user commutes (e.g. "Mon-Fri", "flexible")
org_codeTEXTOrganisation code at time of submission
countryTEXTCountry of the submission (informational, set from form)
journey_statusTEXTactive | expired | archived | deletion_pending
created_atTIMESTAMPTZ DEFAULT now()Submission timestamp
expires_atTIMESTAMPTZSet to created_at + 90 days — submission is excluded from matching after this date
terms_versionTEXTVersion of T&C accepted at time of submission
matches Active
Each row represents a potential carpool pair between two submissions. Created by find-matches. The status field tracks the full lifecycle from creation to mutual confirmation.
ColumnTypeDescription
match_idSERIAL PKAuto-incrementing primary key
sub_a_idINT FK → submissionsFirst submission in the pair
sub_b_idINT FK → submissionsSecond submission in the pair
match_strengthINT (0–100)Compatibility score. Higher = better match. Based on proximity of start/end points and schedule overlap.
statusTEXTnew | notified | viewed | interest_expressed | not_interested | mutual_confirmed | user_deleted
interest_aBOOLEANWhether user A has expressed interest
interest_bBOOLEANWhether user B has expressed interest
interest_a_atTIMESTAMPTZWhen user A expressed interest
interest_b_atTIMESTAMPTZWhen user B expressed interest
notified_atTIMESTAMPTZWhen both users were notified of this match by email
created_atTIMESTAMPTZ DEFAULT now()When the match record was created
organisations Active
Partner organisations that use Community Carpool. Users can identify as belonging to an organisation when submitting a journey.
ColumnTypeDescription
org_idSERIAL PKAuto-incrementing primary key
org_codeTEXT UNIQUE NOT NULLShort code used in URLs and form fields (e.g. ACME)
nameTEXTFull organisation name (e.g. "Acme Corporation")
domainTEXTEmail domain for the organisation (e.g. acme.com) — for future auto-detection
is_activeBOOLEAN DEFAULT trueWhen false, org does not appear in dropdowns or matching logic
support_tickets Active
Support requests submitted via the support page. Ticket type deletion is blocked — users are directed to the dedicated manage-deletion flow instead.
ColumnTypeDescription
ticket_idSERIAL PKAuto-incrementing primary key
emailTEXT NOT NULLSubmitter's email address
typeTEXTreport_user | technical | general_enquiry
messageTEXTThe ticket message content
created_atTIMESTAMPTZ DEFAULT now()Submission timestamp
statusTEXT DEFAULT 'open'open | closed
admin_notesTEXTInternal notes added by admins — append-only, timestamped
closed_atTIMESTAMPTZWhen the ticket was closed
deletion_log Active
Immutable audit log of completed account deletions. Contains metadata only — no personal content (messages, locations, etc.) is retained after deletion.
ColumnTypeDescription
log_idSERIAL PKAuto-incrementing primary key
user_idINTThe user_id of the deleted account (retained for reference)
emailTEXTEmail of the deleted account
deleted_atTIMESTAMPTZWhen the deletion was processed
reasonTEXTReason for deletion (e.g. "user_requested", "admin_action")
submissions_deletedINTNumber of submission records deleted
matches_deletedINTNumber of match records deleted
config Active
Key-value store for platform configuration. All edge functions read from this table for operational settings. See Config table for full details of each key.
ColumnTypeDescription
keyTEXT PRIMARY KEYConfiguration key name
valueTEXT NOT NULLConfiguration value (always stored as text — cast as needed)
blacklist Active
Email addresses that are blocked from submitting journeys. Checked by submit-journey.
ColumnTypeDescription
emailTEXT PRIMARY KEYBlacklisted email address
added_atTIMESTAMPTZ DEFAULT now()When the email was blacklisted
added_byTEXTWho added the entry (admin name or "system")
reasonTEXTReason for blacklisting (e.g. "report_user ticket #42")
admin_users Planned
Admin panel user accounts. Role-based access control. Not yet implemented.
ColumnTypeDescription
admin_idSERIAL PKAuto-incrementing primary key
emailTEXT UNIQUE NOT NULLAdmin email address
nameTEXTDisplay name
roleTEXTsuper_admin | admin | support
password_hashTEXTPBKDF2 hash: saltHex:hashHex (16-byte salt, 100k iterations, SHA-256)
deletion_pinTEXTHashed 6-digit PIN required for destructive actions
is_activeBOOLEAN DEFAULT trueWhen false, login is immediately rejected
role_expires_atTIMESTAMPTZOptional: if set, login fails after this date
allowed_ipsTEXT[]Optional IP allowlist — login fails from other IPs
created_atTIMESTAMPTZ DEFAULT now()Account creation timestamp
admin_sessions Planned
Active admin login sessions. Sessions are validated on every API request.
ColumnTypeDescription
session_idTEXT PKCryptographically random session token
admin_idINT FK → admin_usersThe logged-in admin
created_atTIMESTAMPTZLogin timestamp
expires_atTIMESTAMPTZSession expiry (e.g. 8 hours)
ip_addressTEXTIP address of the session
admin_actions Planned
Immutable audit log of all admin panel actions. Every action is recorded for accountability.
ColumnTypeDescription
action_idSERIAL PKAuto-incrementing primary key
admin_idINT FK → admin_usersWhich admin performed the action
action_typeTEXTe.g. delete_user, close_ticket, add_blacklist
target_typeTEXTe.g. user, ticket, blacklist
target_idTEXTID of the affected entity
notesTEXTOptional additional context
created_atTIMESTAMPTZ DEFAULT now()Timestamp of the action
user_notifications Planned
Per-user notification preferences. Currently email preferences are managed via a separate mechanism in the users table. This table is planned for a more granular notification system.
ColumnTypeDescription
user_idINT FK → users PKOne row per user
unsubscribed_matchesBOOLEAN DEFAULT falseOpt out of match notification emails
unsubscribed_remindersBOOLEAN DEFAULT falseOpt out of reminder emails
unsubscribed_marketingBOOLEAN DEFAULT falseOpt out of marketing emails
updated_atTIMESTAMPTZLast preference update timestamp
